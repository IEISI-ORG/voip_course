# MARP Rendering (E4)

Renders the instructor decks in [`../slides/`](../slides/) (generated by D0) to HTML/PDF and
**validates** that each deck actually renders, using Playwright under a headless X server.

## Requirements
- `marp-cli` (`marp`), `node`, `xvfb-run`.
- Playwright module + a chromium browser: `make deps` (network — installs `playwright` and
  `npx playwright install chromium`).

## Use
```bash
make html      # ../slides/*.md -> out/*.html (self-contained)
make pdf       # -> out/*.pdf (needs a chromium for marp)
make check     # render + Playwright render-check under xvfb (the E4 gate)
make watch     # live preview while editing
bash verify.sh # make html + render-check; SKIPs cleanly if the toolchain is absent
```

## What the check proves
`render-check.js` opens each rendered deck in headless chromium (under `xvfb-run`), asserts it
contains MARP slides, and fails on any page/console error — so a deck that builds but renders
broken is caught. Validated here: 20 decks, all PASS.

## Notes
- `out/`, `node_modules/`, and `package-lock.json` are build artifacts (gitignored). The decks
  themselves are generated from the module docs — edit the module, run D0's `build-slides.sh`, then
  re-render here.
- CI runs shell/YAML lint and the offline graders; this render step runs where the marp+browser
  toolchain exists (like the Docker-dependent lab graders), not in the lint-only CI job.
